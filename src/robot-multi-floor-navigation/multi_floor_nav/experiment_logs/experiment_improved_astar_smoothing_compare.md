# Improved A* 平滑前后对比实验记录

本文档用于独立记录 **Improved A*** 在“关闭平滑 / 开启平滑”两种配置下的规划性能差异。该实验的目标是评估**规划算法与平滑模块本身**的收益，因此应与 SP-AMCL 扰动恢复实验分离，避免将定位误差变化混入规划对照。

## 1. 实验目的

- 比较平滑前后路径几何质量是否改善。
- 比较平滑前后安全性指标是否改善。
- 比较平滑前后规划耗时与额外平滑耗时是否可接受。
- 在固定地图、固定任务、固定定位前提下，给出规划算法层面的公平对照结论。

## 2. 变量控制原则

### 2.1 需要固定的条件

- **地图**：保持一致。
- **起点 / 终点任务**：保持一致。
- **机器人模型与 costmap 配置**：保持一致。
- **定位前提**：保持一致。
- **唯一对照变量**：是否开启路径平滑。

### 2.2 与 SP-AMCL 扰动实验的关系

本实验**不用于**评估 SP-AMCL 在 `none / small / medium / large` 扰动下的恢复能力。

同时，为减少定位与进电梯过程对规划对照的干扰，本实验统一采用**传统 AMCL** 作为定位方案，不启用 `SP-AMCL` 区域初始化。该选择参考了历史稳定版本 ` /home/eethanhe/第五章联合仿真/联合仿真载体/multi_floor_nav ` 的实验组织方式：该版本在进电梯与定位链路上更稳定，适合作为规划算法对比时的定位基线。

若论文需要展示定位模块鲁棒性，应引用独立文档：

- `experiment_none_perturbation_spamcl.md`
- `experiment_small_perturbation_spamcl.md`
- `experiment_medium_perturbation_spamcl.md`
- `experiment_large_perturbation_spamcl.md`

若论文需要展示平滑收益，应只引用当前文档中的对照结果。

## 3. 实验配置

### 3.1 推荐实验设置

建议优先采用**无扰动 + 传统 AMCL** 开展平滑对照，确保比较对象仅为规划算法本身。

| 配置项 | 建议值 | 说明 |
| --- | --- | --- |
| 定位方案 | 传统 `AMCL` | 与参考稳定版本保持一致，不启用 `SP-AMCL` |
| 扰动场景 | `none` | 避免扰动对规划结果产生额外影响 |
| 地图 | 固定楼层地图 / 固定全流程任务 | 与论文主实验保持一致 |
| 规划器 | Improved A* | 对照对象一致 |
| 对照开关 | `enable_path_smoothing=false/true` | 唯一核心变量 |
| 重复次数 | ≥ 3 次 | 降低偶然性 |

### 3.2 运行脚本

当前仓库已有脚本：`scripts/amcl/run_improved_astar_smoothing_compare.sh`

该脚本现已默认采用 `baseline` 方法，即**传统 AMCL** 定位，不再走 `SP-AMCL`。同时，脚本会启用**参考版进电梯行为**，使进入电梯阶段尽量对齐历史稳定版本中的固定速度、固定距离流程，减少额外安全分支对规划对照的干扰。若本实验仅关注规划平滑对比，建议优先使用 `none` 场景，使“定位方案 + 扰动设置 + 进电梯行为”尽量与参考稳定版本保持一致。

```bash
cd /home/eethanhe/catkin_ws
```

### 5.3 基于本轮数据的分析

1. **几何质量显著改善（核心收益）**
   - `waypoints` 均值由 `128.71` 降至 `4.06`，降幅约 `96.85%`。
   - `turn_point_count` 均值由 `11.79` 降至 `0.92`，降幅约 `92.20%`。
   - `sum_abs_turn_rad` 均值由 `11.300` 降至 `1.125`，降幅约 `90.04%`。
   - `path_length_m` 均值由 `2.766` 降至 `1.792`，降幅约 `35.21%`。

2. **安全性未劣化（约束满足）**
   - `min_clearance_m` 保持 `0.260 -> 0.260`，未下降。
   - `mean_clearance_m` 保持 `0.261 -> 0.261`，未下降。
   - `danger_point_count` 始终为 `0`，未引入新的危险点。

3. **搜索代价基本不变（符合机理）**
   - `expanded_nodes` 均值 `1172.99 -> 1174.50`，仅约 `+0.13%`，可视为同量级波动。
   - 该现象与算法流程一致：平滑主要作用于搜索后的路径几何，不改变 A* 主搜索空间规模。

4. **稳定性结论**
   - 本轮窗口内成功规划调用次数一致（`1215/1215`），失败次数均为 `0`。
   - 结合进电梯链路本轮全流程成功，可认为在当前 `none + baseline` 条件下，平滑模块带来的是“几何收益”，而非以稳定性或安全性为代价。

## 6. 论文写作建议

### 6.1 本节建议回答的问题

- 平滑后路径是否更短、更顺。
- 平滑后是否减少了不必要折线与剧烈转向。
- 平滑后是否保持或提升安全间隙。
- 平滑带来的额外耗时是否在系统可接受范围内。

### 6.2 建议表述模板

**本实验旨在独立评估 Improved A* 路径平滑模块的收益。** 在固定地图、固定任务与固定定位前提下，对比关闭平滑与开启平滑两种配置可见：平滑后 `path_length_m` 均值下降约 `35.21%`，`turn_point_count` 均值下降约 `92.20%`，`sum_abs_turn_rad` 均值下降约 `90.04%`，同时 `min_clearance_m` 与 `mean_clearance_m` 基本不变、`danger_point_count` 保持为 `0`。该结果表明平滑模块显著提升了路径几何质量与可执行性，且未引入可观测的安全性退化；若后续补充耗时统计仍在可接受范围内，可将其作为论文中“规划后处理有效性”的直接证据。
