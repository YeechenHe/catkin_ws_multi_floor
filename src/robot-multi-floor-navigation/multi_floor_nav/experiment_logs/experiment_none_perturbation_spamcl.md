# None 扰动 + SP-AMCL 全流程导航实验记录

本文档整理单次仿真实验的配置参数、过程指标与落盘数据，便于论文引用与复现。

## 1. 实验概述

- **场景**：多楼层仿真导航（Gazebo + ROS Noetic），启用 **模块 A（SP-AMCL 区域初始化）**。
- **扰动等级**：None —— 在 **L1 切图后** 对 `initialpose` 不施加扰动（L0 同样无扰动）。
- **论文/报告关注点**：验证无扰动基线条件下，SP-AMCL 不退化且全流程稳定。
- **全局规划器**：Improved A*（`move_base` 插件）。
- **定位**：AMCL + SP-AMCL 分层参数（`spa_amcl_region_init.yaml`）。

### 1.1 本轮归档实验（已回填）

- **启动方式**：`roslaunch multi_floor_nav nav.launch enable_module_a:=true gui:=true headless:=false`，另终端 `rostopic pub -1 /start std_msgs/Empty "{}"`。
- **L1 重定位 CSV（主数据）**：`experiment_logs/reloc_curve_1775724452_L1.csv`（38 个采样点，完整内容见 **§3.4**）。
- **主文曲线图（L1）**：`experiment_logs/reloc_curve_1775724452_L1_nom.png`（`plot_reloc_recovery_curve.py`，`linear_error_nom` / `angular_error_nom`，线门限 0.45 m、角门限 0.15 rad）。
- **L0 落盘（过程数据）**：同次运行另生成 `reloc_curve_1775724366_L0.csv` 与 `reloc_curve_1775724366_L0_nom.png`。
- **Improved A* 统计来源**：同次运行的 `roslaunch` 终端捕获。

#### 3.2.2 规划统计使用边界说明

本节中的 `waypoints`、`expanded_nodes`、成功/失败次数等数据，仅用于说明本次 **SP-AMCL 扰动实验** 在全流程导航期间的规划运行背景，证明系统在该无扰动基线工况下未出现规划失效。

若需开展 **Improved A* 平滑前后对比**，应使用独立实验文档 `experiment_improved_astar_smoothing_compare.md`，在固定地图、固定任务与固定定位前提下单独比较路径几何质量、安全性与时延指标，避免将定位扰动因素混入规划算法对照。

### 3.3 结果分析（论文可用表述模板）

在 none 扰动下，理论上 L1 初始误差应接近零。本工况主要用于验证系统在无偏差输入下不发生性能退化，作为 small / medium / large 扰动结果的基准对照。
